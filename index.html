Frequently Asked Questions

1. What's the difference between `var`, `let`, and `const`?

`var` is function-scoped and can be redeclared and updated. It's also hoisted to the top of its scope. `let` is block-scoped, can be updated but not redeclared within the same scope, and is not hoisted. `const` is also block-scoped but cannot be updated or redeclared, and must be initialized at declaration. Objects and arrays declared with `const` can have their properties modified, but the variable itself cannot be reassigned.

2. What's the difference between `map()`, `forEach()`, and `filter()`?

`map()` creates a new array by applying a function to each element of the original array and returning the results. `forEach()` executes a function on each element of an array but doesn't return anything. `filter()` creates a new array containing only the elements that pass a specified condition. In short: use `map()` for transforming elements, `forEach()` for side effects without creating a new array, and `filter()` for selecting specific elements.

3. How do arrow functions differ from regular functions?

Arrow functions have a more concise syntax and automatically bind `this` to the surrounding context, unlike regular functions which have their own `this` context. Arrow functions don't have their own `arguments` object and can't be used as constructors (no `new` keyword). They're also not hoisted and can't be used for methods when `this` should refer to the object. Arrow functions are ideal for callbacks and when you want to preserve the lexical scope.

4. How do JavaScript Promises work?

Promises are objects representing the eventual completion or failure of an asynchronous operation. They have three states: pending, fulfilled, or rejected. A Promise takes a function with two parameters: resolve and reject. You can chain Promises using `.then()` for successful outcomes and `.catch()` for errors. The `.finally()` method executes regardless of success or failure. Promises help manage asynchronous code more elegantly than callbacks, avoiding "callback hell" and allowing for better error handling. Multiple Promises can be managed with methods like `Promise.all()`, `Promise.race()`, and `Promise.allSettled()`.

5. How do closures work in JavaScript?

Closures occur when a function retains access to its lexical scope even when executed outside that scope. When you define a function inside another function, the inner function has access to the outer function's variables and parameters, even after the outer function has finished executing. Closures are commonly used for data privacy (creating private variables), factory functions, and maintaining state in callbacks. They're fundamental to JavaScript's module pattern and enable powerful programming patterns but should be used carefully as they can lead to memory leaks if variables are unnecessarily preserved.
